# Phase 1 Data Model: Nix Flake — Dev Shell, Package, and App

**Feature**: `002-flake-nix-devshell` | **Date**: 2026-06-03

This feature has no runtime data. The "entities" are the flake's declarative inputs and outputs and
the files that hold the pins. Modeled here so the contract and tasks stay consistent.

## Entities

### Flake (`flake.nix`)

The single declarative entry point at the repo root.

| Field | Value / Type | Notes |
|-------|--------------|-------|
| `description` | string | e.g. "shap — shell-native interface for coding agents". |
| `inputs.nixpkgs` | flake ref | `github:NixOS/nixpkgs/nixpkgs-unstable`. |
| `inputs.rust-overlay` | flake ref | `github:oxalica/rust-overlay`, `inputs.nixpkgs.follows = "nixpkgs"`. |
| `inputs.llm-agents` | flake ref | `github:numtide/llm-agents.nix`; does **not** follow nixpkgs (D5). |
| `outputs` | function | Produces `devShells`, `packages`, `apps` per supported system. |

**Validation**: `nix flake check` evaluates; `nix flake metadata` lists locked inputs.

### SupportedSystems

| Field | Value |
|-------|-------|
| set | `[ "aarch64-darwin" "x86_64-darwin" "aarch64-linux" "x86_64-linux" ]` |

**Rule**: every output attrset is keyed by exactly these systems (FR-008). A system outside the set has
no attribute → evaluation error naming the missing system.

### RustToolchain (`rust-toolchain.toml`)

Single pinned toolchain, shared by the dev shell and the package (FR-002, FR-013).

| Field | Value | Notes |
|-------|-------|-------|
| `toolchain.channel` | `"1.88.0"` | Tracks `workspace.package.rust-version`. |
| `toolchain.components` | `["clippy" "rustfmt" "rust-src"]` | Lint, format, std sources for rust-analyzer. |

Resolved in-flake as `rustToolchain = pkgs.rust-bin.fromRustupToolchainFile ./rust-toolchain.toml`.

### DevShell (`devShells.<system>.default`)

| Field | Type | Contents |
|-------|------|----------|
| `packages` | list | `rustToolchain`, `cargo-nextest`, `cargo-deny`, `rust-analyzer`, `fzf`, `git`, `codex-acp`, `claude-agent-acp`. |
| `env.RUST_SRC_PATH` | path | From `rustToolchain` so rust-analyzer finds std. |

**Rule**: build/test/format/lint succeed using only these (FR-003); ACP adapters + fzf + git are
optional extras and never gate build/test (FR-004).

### Package (`packages.<system>.{shap,default}`)

| Field | Value | Notes |
|-------|-------|-------|
| `pname` | `"shap"` | |
| `version` | `"0.1.0"` | Tracks `workspace.package.version`. |
| `src` | repo root | The Cargo workspace. |
| `cargoLock.lockFile` | `./Cargo.lock` | Consumes committed lock; fails on mismatch (FR-011). |
| toolchain | `makeRustPlatform { cargo = rustToolchain; rustc = rustToolchain; }` | Same pin as dev shell (FR-006, FR-013). |
| output | `$out/bin/shap` | Binary name from `crates/shap-cli` `[[bin]] name = "shap"`. |

`packages.default` aliases `packages.shap` (FR-010).

### App (`apps.<system>.{shap,default}`)

| Field | Value |
|-------|-------|
| `type` | `"app"` |
| `program` | `"${packages.<system>.shap}/bin/shap"` |

`apps.default` aliases `apps.shap` (FR-007, FR-010).

### Lock (`flake.lock`)

Committed record of exact locked revisions for `nixpkgs`, `rust-overlay`, `llm-agents` (and transitive
inputs). Generated by the first `nix` evaluation; committed so all consumers resolve identical inputs
(FR-009, SC-004).

## Relationships

```
rust-toolchain.toml ──fromRustupToolchainFile──> rustToolchain
                                                   │
                       ┌───────────────────────────┴───────────────┐
                       ▼                                            ▼
                  DevShell.packages                      makeRustPlatform → Package
                       ▲                                            │
   llm-agents.packages.${system}.{codex-acp,claude-agent-acp}      ▼
   nixpkgs {cargo-nextest,cargo-deny,fzf,git,...}            App.program → bin/shap

Cargo.lock ──cargoLock.lockFile──> Package        flake.lock pins {nixpkgs, rust-overlay, llm-agents}
```
